The filter you can't see is a filter that doesn't exist.

A real owner went looking for the Medication filter on History and never found it — it sat 8th of 9 chips inside a hidden-overflow horizontal scroll. The edge-fade "there's more" cue (B-146's one carve-out) failed in practice.

The fix: the event-type filter becomes the same quiet pill + bottom sheet as the date scope — one shared ScopeMenu primitive, two pills side by side. Every option is a full-width sheet row. Nothing can hide.

Not a proposal — this mirrors the code shipped in PR #421

What she was up against

The shipped rail, drawn honestly: only 4½ chips fit a 320-pt frame. Everything to the right exists — with no scrollbar, no count, and a 28-px fade as the only hint. "Medication" is four chips past the edge.

The three rules this locks in

No option ever hides

The B-146 bar, now without exceptions: a closed single-select set renders every option on screen. Sheets scroll visibly inside themselves; nothing sits past an invisible fold.

Filtering is always legible

Any non-default scope tints its pill (accent border + colorAccentLight ground — the same active language as FilterChip). Applies to type, presets, and the day drill-in alike.

ChipGroup vs ScopeMenu

Options that are the screen's content (capture forms) wrap in-line via ChipGroup. Options that are a lens over content (list filters) go behind a pill via ScopeMenu. Documented in the component.
