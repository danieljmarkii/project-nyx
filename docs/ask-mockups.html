Culprit — design exploration · 2026-07-18

Ask — answers from your pet's own record

Owner-initiated Q&A over this pet's logged data. Not a chatbot: answer-first cards built from the app's own components, chips before keyboard, every number carrying its denominator and a tap-through to the source events. The model plans and phrases; the deterministic query layer computes. Escalation never comes from the model — it can only relay what the Signal engine already fired.

Answer card The core pattern

Newsreader answer line (the Signal's voice), the app's own calendar pips, the denominator, and a provenance tap-through to filtered History. Follow-up chips continue in-session; nothing persists as a transcript.

The model never does arithmetic Every number is computed by the deterministic query layer (the same code family as Patterns/Signal) and slotted into the card; the model classifies the question and writes the phrasing. One hallucinated stat over an owner's own data is a permanent trust wound (Strava/Notion lesson).
Relay, never mint If a safety finding is live (worsening, chronicity, intake decline), its card leads the answer — the same card Home shows, free. Ask introduces no new escalations and no new reassurance. If the engine is silent, Ask stays silent about wellness.
Voice nyx-voice throughout: specific over generic, no exclamation marks, pet by name, the register of a caring friend who knows veterinary medicine. "I" appears only as the app's quiet working voice ("I'll show my sources"), never a named character.

Notes & photos are in scope — scoped to the question (D2, ratified) "What did my note say on Tuesday?" gets your note back, verbatim. Photos answer via their AI read — cached first, or a live read through the same analyze-vomit path the detail screen uses (stripped transform, shared 10/day read cap, result saved to the event). Never a bulk dump: the question picks the events, and nothing else crosses.
Deleted events stay deleted Every query tool filters deleted_at IS NULL — the B-071 lesson, carried as a hard contract + test fixture ("more deleted than live" pet).

D2 — RATIFIED (PM, 2026-07-18; spec v2.1 §6). The full logged record is in scope — including notes and photos — under three mechanisms that replace field-tier exclusion: scoped retrieval (the question picks the events; no bulk tool exists), transform-only photo access (the report's EXIF/GPS-stripping path, never raw originals), and one read path (photo reads run through the analyze-vomit machinery, so Ask and the detail screen can never disagree about what a photo showed). T&S's tier-2 recommendation and dissent are recorded in the spec; the privacy-policy line becomes "your logged data — including your notes, and photos when your question needs them."

The rundown Ships in v1

Reads like the report's first minute: counts with denominators, no adjectives, no verdicts. "Share the full vet report" hands off to the Step-9 flow — Ask covers the long tail of follow-ups; the report stays the clinical artifact.

Why deterministic An anxious owner in a consult room is the least able user of a blank text box in the product. A preset needs no typing, no model, no network luck — and its numbers are the engine's, so the vet can audit any tile in one tap.
What's deliberately NOT here No proactive voice (Signal owns proactivity — one channel, Principle 4). No persisted chat history; answers are ephemeral, the rundown is pinnable. No night ground (Ask is a records surface — day system; the whorl is the only working moment). No voice input in v1 (backlog). No write tools ever — Ask reads the record, it never edits it.
Open with the name "Ask" as the working label reads clean in the header pill and matches the mental model. Alternatives if you want distance from competitors' "Ask AI": "Rundown," "Q&A," or pet-name framing ("Ask about Pixel"). Your call at spec time.
