History filters — a calmer way to organize

For the product-team discussion on the History tab filters. The screen carries two filter dimensions — a time scope (when) and an event-type lens (what). Today they're two competing rows of differently-styled chips; one clips off-screen and can't scroll. Direction: separate the two by kind of control — scope becomes one quiet menu in the title row, the type lens becomes one clean scrollable row in a single visual language.

Tokens mirror constants/theme.ts (v1.2 "Linear Clean"). Geist body · Newsreader display. Not pixel-final — this settles layout & control choice before any code. Designer + Jordan + Sam.

What this mock settles

The core move

  • Two dimensions, two kinds of control. Scope (when) is a single mutually-exclusive choice with long labels → a quiet menu. Lens (what) is a set you toggle and scan → a row of chips. Today they're both chip rows fighting for the same space; that's the "mess."
  • One visual language for the chips. The black-fill vs teal-outline split goes away — the only chips left are the type lens, in one calm teal-tint selected state (accent = interactive, per the design system; never decorative).
  • Nothing clips, nothing hides. The scope menu can't overflow. The lens row gets an edge fade so "there's more →" is visible at a glance.

Fixes that ride along

  • Label drift. The chip read "Diarrhea" while the rows read "Loose stool." Chips now read from EVENT_TYPES — one source of truth.
  • Missing Medication. medication is loggable today but had no filter. Added.

Persona read

  • DesignerCalm over busy. Removes a competing visual language and a clipping bug; the lens reads as one scannable strip. Benchmark: the way Linear separates a global scope control from in-view filters.
  • Jordan"Show me just the vomits, last 7 days." Vomit chip is right there; scope is two taps in a friendly menu — no cramped, half-cut chip to thumb at. Plain words, no jargon ("Loose stool", not "Diarrhea").
  • SamGrazing cat → reaches for Meal often (visible first), and now Medication for a cat on meds. "Loose stool" matching the rows removes the who-said-what confusion.
  • Open for PM (1): scope control — B (menu sheet, recommended) vs C (inline segmented). Trade-off is friendly-labels-and-quiet vs zero-extra-taps.
  • Open for PM (2): selected-chip style — teal-tint (shown, calmer) vs solid accent fill (stronger "selected"). Trivial to swap.
  • Out of scope / flag: merging diarrhea + stool_normal into one "Stool" lens is the open stool-consolidation question — kept as two so the diet-trial owner can isolate loose stools. Not silently changed here.
