What this mock is settling

  • Principle 3 fit. B keeps the AI Signal as the first content card — chrome sits above the intelligence surface, it doesn't push into it. D shows what "too much" costs.
  • No Pet-tab duplication. The strip carries identity (who am I looking at), not the profile (breed/age/conditions/weight/trial) — that stays the Pet tab's job. B's one slim line is orientation, not a profile.
  • Switcher-ready, not switcher-built. C shows the future multi-pet tap-target (dimmed ⌄). We design the avatar/name as the eventual switcher entry point but don't build switching until multi-pet is scoped.
  • Wordmark. "Project Nyx" placeholder in the Newsreader display face, quiet (tertiary) so it reads as identity, not a banner. Real name swaps in later with no layout change.
  • Open for PM: in B's slim line — breed + age (shown), or something warmer/less static (e.g. "Logging since March", as in D)? Breed/age is reference data that never changes; a status line earns its pixels more often. Worth a look on-device both ways.
