Option C, rendered with the real app theme. Event type shown is Vomit; the same control appears on stool, lethargy, and "other". Meals are excluded.

Every logged time carries one quiet confidence tag, set by which path you take — never a question you have to answer: witnessed you saw it · exact time   estimated you found it, know roughly when · single time   window you found it, only know a range · two times (or "before now")

Step 1 · The two design directions for the top of the flow

Both keep witnessed = one tap (pre-selected, just hit Log). They differ only in how prominent "Found it" is. You leaned toward optimizing for the found case (~65% of incidents), which favors Direction 2.
