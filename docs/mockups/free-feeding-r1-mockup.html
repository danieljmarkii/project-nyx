What rev 2 changed (and what the mockup is asserting)

  • Food with food. Pet-page surface removed entirely; free-feeding now lives only on the food detail screen (edit) + food library (view). The pet-level clinical read is the vet report.
  • History = Option C. Ambient strip (standing context, never an event) + boundary markers at start/stop/switch. No synthetic grazing events — that was a hard no from Dr. Chen/Data.
  • Passive freshness. “Confirmed {date}” in the library, with a tap-to-confirm when you're already there. No push notification (Sam's line) — but it guards against a stale arrangement showing false intake.
  • Badge fixed. Free-choice is now a quiet dot + label, not the oversized filled pill.
  • Set once. Still a standing fact — zero additions to the 10-second quick-log path (Principle 1/2).
  • In Signals, honestly. The detector ingests free-fed food as in-window exposure — but a 24/7-constant food is matched-out and rarely a clean correlate. It earns its place via completeness + boundaries + confounder-capping, not flashy findings.
Multi-pet shared-bowl attribution + quantity recovery remain deferred (spec §0). Open non-blocking nit: whether a months-stale arrangement earns one pull-surfaced (never push) prompt — lean is no for v1.
