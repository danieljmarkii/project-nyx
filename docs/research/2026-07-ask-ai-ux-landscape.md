# Best-of-breed "Ask AI" UX — the mid-2026 landscape

**Date:** July 2026 (research run 2026-07-18)
**Prepared for:** Culprit product team — the Ask feature session (B-228, PR #394)
**Method:** web-research sweep (isolated research agent, ~55 source fetches/searches), focused on (1) health-data AI companions, (2) general best-of-breed ask-AI UX, (3) named patterns beyond the chat transcript, (4) documented failures.
**Status:** point-in-time evidence capture. Product decisions taken from it live in `docs/nyx-ask-requirements.md` v2.0, not here (research-folder contract).

**The one-paragraph state of the art:** by mid-2026 the industry has visibly moved past the raw chat transcript. The best "ask" experiences are **answer-first** (structured cards with real data components, citations, follow-up chips), **scoped** (entered from the thing being asked about, context pre-attached), **scaffolded** (personalized suggestion chips so most sessions involve zero typing), and **grounded** (every claim taps through to the user's own source records). Health-data companions add deterministic escalation ladders and never-diagnose guardrails that live in the *system*, not the disclaimer — a distinction the FDA made expensive for Whoop in 2025. The consensus critique (Wattenberger's ["Why Chatbots Are Not the Future"](https://wattenberger.com/thoughts/boo-chatbots/); NN/g's ["AI Chat Is Not (Always) the Answer"](https://www.nngroup.com/articles/ai-chat-not-the-answer/)) has hardened into practice: **chat is a query language, not a destination.**

## a. Twelve named patterns, with exemplars

1. **Contextual "Ask about this" entry (scoped ask).** The ask begins from the object of curiosity with context pre-attached, not from a global chatbot. Gemini's ["Ask about this screen"](https://www.androidpolice.com/google-gemini-ai-assistant-on-screen-conext-rolling-out-now/); Google Photos' per-photo ["Ask" button](https://blog.google/products-and-platforms/products/photos/ask-button-ask-photos-tips/); Whoop Coach's entries inside Sleep/Strain/Recovery ([Whoop](https://www.whoop.com/us/en/thelocker/new-ai-guidance-from-whoop/)); Superhuman's in-thread Ask AI ([review](https://gmelius.com/blog/superhuman-ai-review)).
2. **Search-bar-as-ask / launcher overlay.** One omnibox for retrieval and questions, answering in place — Raycast Quick AI ([Raycast](https://www.raycast.com/core-features/ai)); Notion Q&A inside search ([Notion](https://www.notion.com/help/guides/get-answers-about-content-faster-with-q-and-a)); Dia's chat-with-your-tabs omnibox ([Product Hunt](https://www.producthunt.com/products/dia-browser)). The ask ability is ambient; nobody "goes to the chatbot."
3. **Scaffolded question chips (zero-typing flows).** Pre-written, personalized prompts. Oura added them because they were the **top feature request** of the Advisor beta ([Oura](https://ouraring.com/blog/oura-advisor/)); Whoop shows per-surface curated prompts; Gemini's Gmail panel generates suggestions from your own inbox ([Google Workspace](https://workspace.google.com/blog/product-announcements/new-ways-engage-gemini-workspace)); Ask Photos auto-answers before you type. shapeof.ai's "Wayfinders" family ([Shape of AI](https://www.shapeof.ai/)).
4. **Answer-first card, not transcript.** A structured, scannable artifact — headed sections, key figures, source chips — with conversation demoted to follow-ups. Perplexity is canonical ([teardown](https://aiuxplayground.com/teardowns/perplexity/output/)); Fitbit Labs' Insights Explorer returned one-shot cards of "trends, summaries, explanations, and illustrative charts" ([9to5Google](https://9to5google.com/2024/10/08/fitbit-labs-testing/)).
5. **Generative UI — the answer renders real components.** Oura Advisor "can directly show your Oura data... in visual chart format" ([Oura](https://ouraring.com/blog/oura-advisor/)); Fitbit generated charts from your metrics ([Android Central](https://www.androidcentral.com/apps-software/fitbit-labs-insight-explorer-test-gemini-start)); infrastructure standardized (Vercel [AI SDK generative UI](https://ai-sdk.dev/docs/ai-sdk-ui/generative-user-interfaces), OpenAI Apps SDK, Google A2UI, MCP Apps — [overview](https://www.telusdigital.com/insights/data-and-ai/article/accelerating-genui-ecosystem-mcp-apps-openai-apps-sdk-and-google-a2ui)). Best practice: render *your existing product components* so the answer looks like the app.
6. **Grounding to the user's own records (tap-through citations).** Notion Q&A "always cites sources" ([Notion](https://www.notion.com/help/guides/unearth-fresh-insights-from-your-personal-knowledge-library-using-q-and-a), [Cybernews](https://cybernews.com/ai-tools/notion-ai-review/)); Granola jumps to the exact call moment ([tl;dv](https://tldv.io/blog/granola-review/)); Monarch Money surfaces the underlying transactions ([x1wealth](https://x1wealth.com/compare/copilot-vs-monarch)); Perplexity's numbered citations set the grammar.
7. **Streaming + progress narration (latency masking).** Token streaming drops perceived wait 55–70% at identical total time; skeletons beat spinners ~30%; >5s operations narrate stages (Ask Photos: "Thinking / Searching / Reviewing") ([Google support](https://support.google.com/photos/answer/15318661), [Institute PM](https://www.institutepm.com/knowledge-hub/streaming-ai-responses), [LogRocket](https://blog.logrocket.com/ux-design/skeleton-loading-screen-design/), [uxpatterns.dev](https://uxpatterns.dev/patterns/ai-intelligence/ai-loading-states)).
8. **Follow-up affordances; transparent memory.** Follow-up chips extend the session (Perplexity; Ask Photos "Tell me more"); memory, where it exists, is inspectable and deletable — Oura's "Memories" ([support](https://support.ouraring.com/hc/en-us/articles/39512345699219-Oura-Advisor)); Whoop's "My Memory" page ([Whoop](https://www.whoop.com/us/en/thelocker/my-memory-whoop/)). Counter-example: Fitbit's Insights Explorer took **no follow-ups at all** — a documented limitation ([9to5Google](https://9to5google.com/2024/10/08/fitbit-labs-testing/)).
9. **The proactive lane is separate from chat.** Whoop Daily Outlook ([Gadgets & Wearables](https://gadgetsandwearables.com/2025/01/24/whoop-daily-outlook/)); ChatGPT Pulse's curated morning cards ([OpenAI](https://openai.com/index/introducing-chatgpt-pulse/), [TechCrunch](https://techcrunch.com/2025/09/25/openai-launches-chatgpt-pulse-to-proactively-write-you-morning-briefs/)); Oura's opt-in check-ins. The ask surface itself stays reactive.
10. **Voice as the hands-busy layer, not the primary interface.** ChatGPT Voice / Gemini Live marketed for occupied-hands contexts ([Gemini Live](https://gemini.google/overview/gemini-live/), [guide](https://toolchase.com/blog/chatgpt-voice-mode-guide/)); Whoop roadmaps voice logging/queries ([Whoop](https://www.whoop.com/us/en/thelocker/inside-look-whats-next-for-whoop-in-2025/)).
11. **Deterministic escalation ladder + honest refusal (health-specific).** Ada Health: eight fixed triage levels, fast red-flag escalation, low-confidence → professional assessment (never defaulting to lowest urgency) ([clinician's guide](https://www.iatrox.com/blog/ai-symptom-checkers-decoded-tech-guide-ada-ubiehealth-dx-gpt), [Aalpha](https://www.aalpha.net/blog/ai-symptom-checker-app-development/)). Oura routes medical asks to professionals ([support](https://support.ouraring.com/hc/en-us/articles/39512345699219-Oura-Advisor)).
12. **Caps and premium placement.** Three placements observed: bundled into membership (Oura, Whoop — no extra cost; [Businesswire](https://www.businesswire.com/news/home/20250331565896/en/Oura-Advisor-an-AI-powered-Personal-Health-Companion-Now-Rolling-Out-to-All-Oura-Members)); **AI as the only paid thing in a free app** (Bevel, $14.99/mo solely for "Bevel Intelligence" — [review](https://neura.health/insight/bevel-health-app-in-depth-review)); AI inside a higher tier (Google Health Premium $9.99/mo — [TechCrunch](https://techcrunch.com/2025/10/27/fitbits-revamped-app-with-gemini-powered-health-coach-rolls-out-to-premium-users/); Dia Pro $20/mo; Notion folding AI into Business — [UserJot](https://userjot.com/blog/notion-pricing-2025-plans-ai-costs-explained)). Cap UX: surface remaining usage near the limit (~80%), name the reset time, frame the cap as a known boundary not an error ([monday.com](https://support.monday.com/hc/en-us/articles/29544502265746-AI-Credits), [Shape of AI](https://www.shapeof.ai/patterns/cost-estimates), [RevenueCat on AI COGS](https://www.revenuecat.com/blog/growth/ai-feature-cost-subscription-app-margins)).

## b. The closest analogs, in concrete detail

### Oura Advisor — the closest analog to Ask
LLM companion over the user's own ring data; all members since 2025-03-31 after a Labs beta ([Businesswire](https://www.businesswire.com/news/home/20250331565896/en/Oura-Advisor-an-AI-powered-Personal-Health-Companion-Now-Rolling-Out-to-All-Oura-Members)). **Entry:** Today-tab "+", the menu, and deep links **from Readiness insight cards** ([Oura](https://ouraring.com/blog/oura-advisor/)). **First-run:** a communication-style picker (conversational vs direct) + check-in frequency — tone is a setting, not an imposed personality. **Answers:** text + **inline charts of the user's actual data** + prompt chips. **Memory/threads:** "Memories" viewable/deletable in Settings; full reset; same-day threads auto-continue; history deletable ([support](https://support.ouraring.com/hc/en-us/articles/39512345699219-Oura-Advisor)). **Safety:** "not a medical device... not intended to diagnose," consult-a-doctor-before-changing-medication, "Advisor can make mistakes." **Pricing:** included in the $5.99/mo membership ([US Mobile](https://www.usmobile.com/blog/oura-advisor-ai/)). **Beta numbers:** 60% weekly usage; 83% found answers reliable; 75% of conversations about sleep/recovery ([Oura](https://ouraring.com/blog/oura-advisor/)).

### Whoop Coach + Daily Outlook — reactive ask and proactive briefing as two separate surfaces
Entries on home + embedded in Sleep/Strain/Recovery with curated prompts ([Whoop](https://www.whoop.com/us/en/thelocker/new-ai-guidance-from-whoop/)). **Deterministic algorithms produce the numbers; the LLM phrases** ([announcement](https://www.whoop.com/us/en/thelocker/whoop-unveils-the-new-whoop-coach-powered-by-openai/)). Daily Outlook is a *separate* morning briefing, not chat ([Gadgets & Wearables](https://gadgetsandwearables.com/2025/01/24/whoop-daily-outlook/)). "My Memory" transparency page. **The regulatory arc:** the FDA's 2025-07-14 warning letter over Blood Pressure Insights established that **"intended use is defined by function, not disclaimers"**; a class action followed 2025-11; resolution came mid-2026 only after Whoop *changed the feature* ([FDA letter](https://www.fda.gov/inspections-compliance-enforcement-and-criminal-investigations/warning-letters/whoop-inc-709755-07142025), [breakdown](https://insider.thefdagroup.com/p/fda-warning-letter-breakdown-whoop), [STAT](https://www.statnews.com/2026/06/23/fda-drops-enforcement-against-wearable-maker-whoop/), [ArentFox Schiff](https://www.afslaw.com/perspectives/longevity-lens/whoop-there-it-fda-warning-letter-now-anchors-class-action-against)). Privacy framing: "OpenAI does not store any conversational data."

### Fitbit Insights Explorer → Google Health Coach — the purest "ask your data, get charts" experiment
Labs test (2024-10 → 2025-03): questions over your Fitbit data returned personalized insights + **illustrative charts**, scoped to a **closed set of query classes** (averages, trends, correlations, min/max, medians, personal bests, comparisons), deliberately **stateless (no follow-ups)**, with a plain inaccuracy warning ([9to5Google](https://9to5google.com/2024/10/08/fitbit-labs-testing/), [Android Central](https://www.androidcentral.com/apps-software/fitbit-labs-insight-explorer-test-gemini-start)). Testing ended 2025-03 ([9to5Google](https://9to5google.com/2025/03/02/fitbit-labs-insights-explorer/)); the capability re-emerged inside the Gemini health coach, now behind **Google Health Premium $9.99/mo** (2026-05 relaunch) ([TechCrunch](https://techcrunch.com/2025/10/27/fitbits-revamped-app-with-gemini-powered-health-coach-rolls-out-to-premium-users/), [Google](https://support.google.com/googlehealth/answer/17068213?hl=en), [blog](https://blog.google/products-and-platforms/devices/fitbit/personal-health-coach-public-preview/)).

**One-liners:** Bevel — free app, conversational analysis is the only paid thing ($14.99/mo) ([review](https://neura.health/insight/bevel-health-app-in-depth-review)). Withings Intelligence — 2025-06 beta in Withings+ ([press](https://media.withings.com/press/press-releases/Withings-Intelligence/Withings_Intelligence-EN.pdf)). Apple Health+ — the grand AI "doctor" repeatedly delayed, then scaled down to piecemeal features (2026-02): even Apple couldn't ship the health chatbot ([MacRumors](https://www.macrumors.com/2025/11/10/ai-apple-health-service-still-coming/), [Livity](https://livity-app.com/en/blog/apple-health-ai-coach-shelved)).

**The pet space specifically: no best-in-class "ask over your own pet's logged data" product exists.** The field is telehealth intake AI (Airvet urgency scoring — [co.vet](https://co.vet/post/ai-vet-tools/)), deterministic symptom checkers (Vet-AI/Joii, "165 algorithms" — [Vet-AI](https://www.vet-ai.com/symptomchecker)), photo screening handing off to televets (TTcare — [TTcare](https://www.aiforpet.com/en)), RAG vet-knowledge chat (AskVet — [EyeLevel](https://www.eyelevel.ai/post/askvet-veterinary-care-rag-technology)), and generic "AI Vet" chat apps. The analogs answer *general* pet questions; none answers "what does *my* pet's data say." A Vetster test found **52% of ChatGPT's pet-health answers incorrect**, and vets documented confident hallucination ([Vetster](https://vetster.com/en/wellness/can-chat-gpt-be-trusted-for-pet-health), [Vet Med & Science](https://pmc.ncbi.nlm.nih.gov/articles/PMC11056198/)).

## c. Anti-patterns — documented failures

1. **Bland pep-talk over real data.** Strava Athlete Intelligence: "a bullshit word salad of stats already conveyed by the existing charts," plus factual errors ([Forbes](https://www.forbes.com/sites/cyrusfarivar/2024/10/12/strava-upsets-fans-with-new-ai-feature-that-promises-insights-but-provides-bland-pep-talks/), [Cycling Weekly](https://www.cyclingweekly.com/news/strava-says-its-new-ai-feature-is-not-a-novelty-but-i-think-its-pointless)).
2. **Sycophancy — lethal in health contexts.** The 2025-04 GPT-4o update rolled back in four days after it "endorsed a user's decision to stop taking their medication"; root cause: optimizing short-term thumbs-up ([OpenAI post-mortem](https://openai.com/index/sycophancy-in-gpt-4o/), [TechCrunch](https://techcrunch.com/2025/04/29/openai-explains-why-chatgpt-became-too-sycophantic/)).
3. **Hallucinated stats over personal data.** Notion Q&A hallucinating on answerable questions ([Slite](https://slite.com/learn/how-to-use-notion-ai)); the pet-health 52% error rate above. One wrong number over a user's *own* data is a permanent trust wound ([MIT Tech Review on AI Overviews](https://www.technologyreview.com/2024/05/31/1093019/why-are-googles-ai-overviews-results-so-bad/)).
4. **AI summaries misrepresenting the source.** Apple Intelligence's false BBC headline; formal complaint; feature paused ([Axios](https://www.axios.com/2025/01/17/apple-ai-news-alerts-fake-headlines), [CNBC](https://www.cnbc.com/2025/01/16/apple-disables-ai-notifications-for-news-in-its-beta-iphone-software.html)).
5. **Paywalling weak insight over the user's own data.** Garmin Connect+ ($6.99/mo "Active Intelligence"): 10k-upvote boycott; "hilariously bad" ([TechRadar](https://www.techradar.com/health-fitness/smartwatches/garmins-new-subscription-ai-feature-is-hilariously-bad-so-far), [Tom's Guide](https://www.tomsguide.com/wellness/smartwatches/garmin-sparks-outrage-with-garmin-connect-subscription-paywall-have-your-say)). The rage formula: *my data + trivial output + your fee*.
6. **Chat-as-default / the bolted-on chatbot.** LinkedIn's suggested-question windows as NN/g's canonical low-value example; Wattenberger on blank-box affordance failure ([NN/g](https://www.nngroup.com/articles/ai-chat-not-the-answer/), [Wattenberger](https://wattenberger.com/thoughts/boo-chatbots/)).
7. **Lost answers.** Dia users can't find previous conversations ([case study](https://medium.com/@nimateea/ux-enhancements-discoverable-chat-history-fixing-ux-friction-fba514282f0b)).
8. **No-follow-up dead ends.** Fitbit's Insights Explorer ([9to5Google](https://9to5google.com/2024/10/08/fitbit-labs-testing/)).
9. **Disclaimer-as-fig-leaf.** The Whoop FDA letter: a "not medical advice" footer cannot launder a feature that functions diagnostically ([FDA Law Blog](https://www.thefdalawblog.com/2025/07/blood-pressure-rising-fda-warning-letter-takes-an-aggressive-approach-on-general-wellness-product/), [ArentFox Schiff](https://www.afslaw.com/perspectives/longevity-lens/whoop-there-it-fda-warning-letter-now-anchors-class-action-against)).
10. **Sparkle-icon magic branding.** ✨ signals "some AI thing" and nothing about function ([NN/g](https://www.nngroup.com/articles/ai-sparkles-icon-problem/), [Google Design research](https://design.google/library/ai-sparkle-icon-research-pozos-schmidt), [Slate](https://slate.com/technology/2025/12/artificial-intelligence-tools-icon-google-gemini-chatgpt-design.html)).
11. **No off-switch, no precise feedback.** Strava users' stated fear was no disable option; the feedback control couldn't say *what* was wrong ([Strava community](https://communityhub.strava.com/strava-features-chat-5/athlete-intelligence-feedback-7587), [Forbes](https://www.forbes.com/sites/cyrusfarivar/2024/10/12/strava-upsets-fans-with-new-ai-feature-that-promises-insights-but-provides-bland-pep-talks/)).

## d. Synthesis offered to the spec (decisions live in `nyx-ask-requirements.md` v2.0)

1. **Deterministic answers, generative routing** — the LLM classifies and phrases; the deterministic layer computes (Whoop's split; Fitbit's closed query classes as the template; Strava/Notion as the cost of one hallucinated stat). Also makes most answers nearly free to serve.
2. **The answer is a card rendering the app's own components** — chart first, sentence second (Oura, Fitbit; the generative-UI ecosystem direction).
3. **Every number taps through to its source events** — the Notion/Granola/Monarch pattern; the single highest-leverage trust affordance, and it converts Ask into navigation that survives the LLM being unavailable.
4. **Chips before keyboard; ship a vet-visit preset** — Oura's top beta request; an anxious owner in a consult room is the least able user of a blank text box in the product.
5. **Safety lives in the answer architecture, not the disclaimer** — the Whoop-FDA lesson + the GPT-4o episode: deterministic escalation outside the LLM; a phrasing layer structurally unable to reassure or diagnose; data-gap honesty over inference.
6. **Stateless answers with follow-ups; saved answers over transcripts; memory only if inspectable** (Fitbit's weakness; Dia's lost-history friction; Oura/Whoop's memory transparency).
7. **No proactive voice for Ask** — the proactive lane is a separate, bounded briefing surface everywhere it works (Whoop, Pulse, Oura); Culprit's Signal already is that lane.
8. **Price the conversation, never the count; label the boundary** — counts/trends/escalation over one's own data stay free (Garmin is the cautionary tale); the conversational layer is the legitimate premium wedge (Bevel; Google Health); caps surfaced near the limit with a named reset, gating only the model call.
