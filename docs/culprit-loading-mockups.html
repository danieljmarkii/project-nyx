B-313 reconciliation & N3 — WhorlSpinner + loading system

A working design mock for PM review before any React Native code is written. Colours, type and geometry use the real theme tokens. The spinners are live — toggle Reduced motion (static frame) and Pause (app-blur behaviour) up top.

Locked rules honoured throughout: carve (crescent is always a cutout) · register (night grounds only where the app works on the pet's behalf) · one interactive accent (teal is the sole tappable/live colour; indigo/lavender are world-only) · no metaphor framing (loading is described mechanically, never analogically) · motion budget (≤1 ambient loop per screen; native-driver transforms; paused on blur; a static frame under reduced-motion).

Part 1 — B-313: the "two months" on Patterns

The symptom count card runs a trailing-30-day window; the calendar below it opens on the named calendar month (spec §8.1). Early in a month, under the same "Vomiting" title, they visibly contradict — and the count's "last month" collides with the calendar's literal "June". Both windows are spec-locked, so this is a labelling decision, not a bug to flip.

The clash: "8, ▲2 more than last month" sits directly above "No vomiting logged in June." Same title, two silent windows → reads as a data bug on the wedge surface.

Option A — label the windows Recommended

Give the count card the explicit window frame it's currently missing ("Last 30 days"), and change the shared delta phrase for the month window from "last month" → "the previous 30 days". The calendar already states "June 2026". Now the two read as two different, complementary views — a responsive rolling KPI and a browsable month archive — not a contradiction. Nothing spec-locked moves; every other trailing-window card (intake rate, top foods) stays consistent.

Touches MetricCard (an optional caption prop, symptom-count only) + dashboardCards.ts (the month prior-window phrase).

Option B — accept side-by-side (no code)

Lean on the labels already present: the calendar says "June 2026", the count says "last month". Cheapest, but both N5b reviewers flagged the clash with those labels present — the count card still shows no window frame on the number itself, so "8 above none-in-June" stays confusing. Documents a decision; fixes little.

Make the count card show June's count too (here: 0, "▲ vs May"). The two "Vomiting" cards now agree — but the symptom count desyncs from every other card (intake rate, top foods, protein, composition are all trailing-30), the responsive rolling read is lost (a symptom yesterday won't show until it's "this month"), and the Week/Month/3-Month detail screen is inherently trailing. Part 2 — N3: the WhorlSpinner & loading system (spec §5–§6)

One brand loading mark across three tiers. Four concentric arc ridges counter-rotating at 9 / 14 / 19 / 25s (alternating direction) with a core dot breathing at 2.6s. Two grounds: day (teal + indigo-lavender, for light content surfaces) and night (teal + moonlight, for night grounds). Two sizes: sm (20–24px, inline) and md (44–56px, in-place block).

Note for the theme PR: night ridges reuse shipped tokens (colorAccent + colorMoonlight). The day-ground lavender ridge is the one genuinely new value (#A6A2CE, already the shipped colorTextOnNightMuted — a world/lavender, never interactive, so the one-accent rule holds). I'll tokenise it as colorWhorlRidgeDay rather than hardcode. Flag if you'd rather it be pure-teal (both ridges teal) instead.

Tier 1 — skeletons (<~1s)

Content-shaped shimmer on content surfaces (Patterns cards, History, detail). No Whorl at this duration — the shape of what's coming reads faster than a spinner.

Tier 2 — WhorlSpinner in place (~1–10s) & pull-to-refresh sky

Every bare ActivityIndicator (~25 sites) becomes a WhorlSpinner, sized by expected duration (default sm). Pulling Home reveals a ~112pt night band — stars + WhorlSpinner sm night + one muted line. Zero-text variant is the pre-approved fallback if the line reads busy on-device.

Tier 3 — the night moment (§6): a full-screen wait becomes a branded screen

When the app has nothing else to show yet — cold-start hydration, vet-report generation, a photo read — the wait itself is the night: aurora + starfield, the Whorl at medium-large scale as a background texture (0.9–1.1× width, 40–55% stroke opacity; core dot stays full-opacity — the one focal point), one warm line lower-third. Trigger = full-screen blocking wait AND >~2s AND real work on the pet's behalf. (Exact scale/opacity inside the specced ranges is an on-device tuning AC — locked at N3 QA on your phone.)

The three qualifying call sites & copy (§9)
  • Cold start — “Catching up on {pet}'s history…” / “This only takes a moment.”
  • Vet report — “Building {pet}'s report…” / “Pulling together the full record.”
  • Photo read — “Reading the label…” / “A few seconds.”
Never the night moment: card loads (skeletons), pull-to-refresh / saves / retries (spinner), anything under ~2s. Minimum hold 600ms; dissolves to the destination over ~700ms. ColdStartOverlay rebuilds onto this.
Reduced motion: the whole composition is static (arcs at rest, dot still) — the copy carries the moment. Every tier defines its static frame; the toggle up top shows all of them at once.
